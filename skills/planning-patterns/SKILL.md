---
name: planning-patterns
description: Create phased implementation plans with risks and tests.
---

# Role
Oracle or Builder

# Trigger
- When a task requires a plan before execution.

# Inputs
- Requirements and constraints.

# Outputs
- Phased plan with deliverables, tests, and risks.

# Steps
1. Break scope into phases (Discovery, Architecture, Development, Verification).
2. List tasks per phase with dependencies.
3. Identify risks and mitigation steps.
4. Define verification checkpoints and exit criteria.
